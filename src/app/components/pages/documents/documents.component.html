Why it's important to have a clear mind.
You'll make enough mistakes as a developer. Some say that programming is 10% writing code and 90% fixing the crap you wrote. Why compound that with the wrong environment? Find your favorite music track, light your favorite candle, pour a cup of coffee, plant yourself on your balcony...Whatever it takes to help cultivate your learning experience.
Your Development Enviromnent
IDE
An Integrated Development Environment (IDE) is nothing more than a sandbox to write and test code. IDEs are to web development as powerpoint is to presentations. The most popular IDEs include Visual Studio Code, ATOM, Sublime, and Brackets. There's no clear cut "performance" leader as most have similar features. That said, Visual Studio Code is surging ahead of the competition when it comes to community and distributer support. My courses use Visual Studio Code and a browser based solution called JSFiddle. JSFiddle allows us to quickly write and test code from multiple JS librbaries without installing the packages ourselves.
Browsers
In order to view our code, we need a browser such as Chrome, Safari, Firefox, or Opera. For this course, we will be using Chrome. Chrome has amazing distributer and community support and it's popularity has soared past the competition over the years. Firefox has amazing tools specific the web developers that come native to the browser. Safari and Opera are capable options but they are rarely used as a primary browser by developers. Different browsers render code differently so while it's completely your preference, I'd stick to either Chrome or Firefox.
Third Party Tools
There are a number of add-ons that help make coding much simpler. An overwhelming majority of those packages deal with making JavaScript easier to write and shortens the learning curve. Some of the most popular 3rd party libraries include jQuery, jQuery UI, jQuery Mobile, Boostrap, Animate.css, Underscore.js, Lodash, Angular, React, Vue, Node.js, Express, Koa, and much more.
